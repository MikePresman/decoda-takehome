# Figma Design Prompt

Use this prompt in Figma after uploading Decoda reference screenshots from `design/ai/screenshots/`.

```text
Design a polished healthcare analytics web app for “Beauty Med Spa” that clearly matches the visual style of the uploaded Decoda reference screens.

Goal:
Create a modern, professional patient dashboard for a medical spa. The product should feel calm, credible, premium, and operationally useful for staff and management. Match the uploaded Decoda styling cues closely, especially layout discipline, typography tone, spacing rhythm, card treatments, color balance, and overall product polish.

Brand/style direction:
- Use the uploaded Decoda screenshots as the primary visual reference
- Keep the UI clean, structured, and professional rather than playful
- Make it feel like a real internal product used daily by front desk staff and managers
- Prioritize clarity, information hierarchy, scanability, and trust
- Avoid generic startup-dashboard styling
- Avoid overly dark themes, neon colors, or consumer-app aesthetics

Design requirements:
- Desktop-first web app, with responsive behavior considered
- Create a cohesive app shell with sidebar or top navigation
- Design for dense but readable operational data
- Use strong hierarchy for headings, KPIs, filters, and table content
- Include realistic spacing, empty states, hover states, and loading states
- Use cards, tables, filters, and charts in a way that feels intentional and production-ready

Pages to design:

1. Patient Table Page
Create a patient management page optimized for front desk staff.
Include:
- Page title and short supporting description
- Search bar
- Filter controls
- Sort controls
- Patient table with useful columns such as:
  - Patient name
  - Contact info
  - Source
  - Created date
  - Appointment count
  - Last appointment
  - Lifetime value
  - Status or payment-related signal if useful
- Pagination controls
- Empty state and no-results state
- Optional quick summary cards above the table

The table should feel highly usable, easy to scan, and operationally practical.

2. Analytics Dashboard Page
Create an executive/manager dashboard focused on business insights.
Include:
- KPI summary cards
- Revenue-related metrics
- Patient acquisition/source breakdown
- Top services
- Busiest providers
- Appointment trends or status breakdown
- A layout that balances metrics, charts, and ranked lists
- Charts should feel clean and presentation-ready, not overly decorative

This page should help a med spa owner quickly understand performance, patient behavior, and operational trends.

UI expectations:
- Use realistic healthcare/admin dashboard patterns
- Show polished filter components, table styling, and chart containers
- Maintain consistency across both pages
- Use believable sample data
- Keep accessibility and legibility in mind
- Favor a premium SaaS/health-tech feel aligned to the uploaded Decoda examples

Deliverables:
- One desktop design for Patient Table
- One desktop design for Analytics Dashboard
- Shared design system direction across both
- Optional responsive/mobile adaptations if space allows

Important:
Base the visual language on the uploaded Decoda screenshots, not on generic dashboard defaults.
```
